💻 שלב 2.1: ביצוע ה-Exploit והשגת Webshell (Initial Access)
לאחר שזיהיתי את גרסת ה-CMS הפגיעה, עברתי לשלב הפריצה האקטיבית כדי להשיג דריסת רגל (Foothold) במערכת.

הפעולה שביצעתי:
השתמשתי בסקריפט האקספלויט כדי להעלות קובץ PHP זדוני בשם shell.phar לשרת. קובץ זה משמש כ-Webshell – "דלת אחורית" שמאפשרת לי לשלוח פקודות למערכת ההפעלה דרך פרוטוקול HTTP.

למה נפתח לי הטרמינל השחור (ה-Webshell)?
הסיבה: ניצלתי חולשת RCE (הרצת קוד מרחוק). האפליקציה "חשבה" שהיא מעלה קובץ תמונה תמים, אך בפועל היא שמרה בשרת סקריפט שמקשיב לפקודות שלי.

המטרה: להשיג ממשק שליטה נוח. ה-Webshell (מסוג p0wny@shell) יוצר סימולציה של טרמינל בתוך הדפדפן, מה שחוסך ממני את הצורך להזריק פקודות בצורה ידנית ומסורבלת דרך שורת הכתובת (URL).

💡 למה אני משתמש בזה ובשביל מה?
הסיבה: ללא ה-Webshell, אין לי דרך לתקשר עם מערכת ההפעלה של שרת ה-Dreaming. זהו הצינור שדרכו אני מעביר פקודות ומקבל פלט.

#### **1. כניסה לממשק הניהול (Authentication)**
**מה עשיתי:** ניגשתי לדף הלוגין וניסיתי להשתמש בפרטי ברירת מחדל שמצאתי באינטרנט: `admin:password`.
* **הסיבה:** מערכות רבות מותקנות ללא שינוי סיסמת המנהל, מה שמהווה "דלת פתוחה" לתוקף.
* **המטרה:** להשיג הרשאות ניהול בתוך האפליקציה כדי לגשת לתכונות מתקדמות (כמו העלאת קבצים).

#### **2. ניצול חולשת RCE (Remote Code Execution)**
**מה עשיתי:** השתמשתי באקספלויט ידוע (CVE-2020-29607) המנצל פגם במנגנון העלאת התמונות של Pluck.



* **הפקודות שביצעתי:**
    ```bash
    # הורדת הסקריפט ממאגר Exploit-DB
    wget https://www.exploit-db.com/download/49909 -O exploit.py

    # הרצת האקספלויט להעלאת Webshell (קובץ PHP זדוני)
    python3 exploit.py <MACHINE_IP> 80 password /app/pluck-4.7.13
    ```
* **הסיבה:** בגרסה זו, המערכת לא מסננת נכון קבצים המועלים לשרת, מה שמאפשר להעלות קוד במקום תמונה.
* **המטרה:** להשיג **Webshell** – ממשק שדרכו אוכל להריץ פקודות לינוקס ישירות על השרת דרך הדפדפן.

#### **3. סריקה פנימית ומציאת פרטי Lucien (Enumeration & Lateral Movement)**
**מה עשיתי:** השתמשתי ב-Webshell כדי לחקור את תיקיית `/opt` ומצאתי קובץ בשם `test.py`.
* **הפקודה שביצעתי (דרך ה-Webshell):** `ls -l /opt` ולאחר מכן `cat /opt/test.py`.
* **מה גיליתי:** מצאתי סקריפט שבו הופיעה הסיסמה של המשתמש **Lucien** בטקסט גלוי (Cleartext).
* **הסיבה:** כמשתמש של שרת האינטרנט (`www-data`), אני מוגבל מאוד. אני חייב למצוא פרטי משתמש אמיתי כדי להתקדם במערכת.
* **המטרה:** לעבור מגישת דפדפן זמנית לגישת **SSH** יציבה.



#### **4. התחברות ב-SSH וקריאת הדגל**
**מה עשיתי:** השתמשתי בסיסמה שמצאתי כדי להתחבר למכונה כמשתמש Lucien.
* **הפקודה שביצעתי:** `ssh lucien@<MACHINE_IP>`
* **התוצאה:** החיבור הצליח. כעת אני מחובר כמשתמש Lucien, מה שאיפשר לי לקרוא את קובץ הדגל הראשון (`user.txt`).

---

**בקיצור:** השתמשתי בסיסמת ברירת מחדל כדי להיכנס ל-CMS, ניצלתי חולשת העלאת קבצים (RCE) כדי להריץ פקודות, ומצאתי סקריפט שנשכח בשרת עם סיסמה של משתמש אחר, מה שנתן לי גישה מלאה למכונה דרך SSH.
